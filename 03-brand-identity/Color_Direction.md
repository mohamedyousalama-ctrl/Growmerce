# Color Direction

> Colour principles and system roles. This document does **not** pick final hex values — it defines
> the *roles* colour must play and the *principles* that govern them, so the palette expresses the
> strategy rather than following a trend.

---

## 1. The governing principle

> Colour serves **meaning and calm**, never decoration or hype. In a restrained, premium,
> intelligence-led identity, colour is used **purposefully and sparingly** — most of the interface
> is calm neutral space, and colour appears where it carries information (confidence, risk, growth,
> action, evidence).

The palette should feel like a serious instrument's colour language, not a marketing palette.

---

## 2. What the palette must support

The colour system must encode commerce-intelligence meaning, not just look nice. It must express:

- **trust** · **intelligence** · **sales growth** · **confidence** · **warnings** ·
  **opportunities** · **evidence** · **priority** · **risk** · **profit/revenue distinction**

This is unusually demanding: the palette is a *semantic system*, because the product reasons about
exactly these dimensions (20). Colour is part of how the intelligence is communicated.

---

## 3. Colour system roles

Rather than "a brand colour + accents," define **functional roles** (Claude Design assigns actual
values to each):

| Role | Purpose | Principle |
|---|---|---|
| **Base** | the dominant calm ground (primarily light) | spacious, neutral, premium; carries most of the UI |
| **Surface** | cards, panels, command surfaces | subtle separation from base; quiet, not heavy |
| **Text** | primary/secondary/tertiary text | high legibility, clear hierarchy, Arabic-optimised contrast |
| **Brand / primary** | the core brand colour; intelligence/trust | confident, serious, premium; used with restraint |
| **Growth** | revenue/sales-growth signals, positive movement | distinct from "profit"; not a cheap/loud green |
| **Profit / margin** | the profit dimension (distinct from growth/revenue) | a deliberate, separate signal — the H1 distinction made visual |
| **Confidence / evidence** | confidence levels and proof strength | a calm gradient/scale, not alarmist; the trust language |
| **Action / priority** | recommended actions, "do this first" | clear, decisive, but not hype; guides without shouting |
| **Risk / warning** | threats, risk, caution | serious and legible, never loud/alarming red theatre |
| **Neutral data** | charts, metrics, structural elements | quiet, precise, non-distracting data colours |

> Note the deliberate **profit vs. growth** split: because Growmerce separates revenue from profit
> (Pillar 2, belief 1), the *colour system itself* should be able to distinguish them. This is a
> rare, strategy-driven requirement and a differentiator.

---

## 4. The confidence/evidence colour scale (a core differentiator)

Confidence and evidence strength need their own visual language (`Trust_and_Confidence_Visual_Language.md`).
Colour principle:
- A **calm, legible scale** that distinguishes Low / Medium / High / Very High confidence without
  alarmism.
- It should read as *honest calibration*, not red-amber-green "good/bad" theatre.
- Evidence strength (tiers) and missing-data states need quiet, distinct treatments.
- This scale must work in Arabic-first, RTL contexts and at small/mobile sizes.

---

## 5. Principles for the whole palette

- **Mostly neutral.** The interface is predominantly calm base/surface/neutral; semantic colour is
  the exception, used where it means something.
- **Light-primary, not dark-only.** A calm, premium, primarily light identity (a considered dark
  mode may exist, but the brand is not a dark dashboard).
- **Restraint over vibrancy.** Muted, sophisticated, premium colour — never neon or saturated hype.
- **Meaning is consistent.** Each role's colour means the same thing everywhere (growth always
  reads as growth, risk as risk) — a learnable, trustworthy system.
- **Accessible contrast.** Especially for Arabic text and for data/metrics legibility.
- **Premium, timeless.** Avoid trend palettes that will date; aim for enduring sophistication.

---

## 6. What to avoid (hard no's)

| Avoid | Why |
|---|---|
| **Neon / glowing colour** | "AI startup" / hype; contradicts calm |
| **Cheap, loud green** | "sales!" cliché; undercuts premium/seriousness |
| **Loud, alarmist red** | risk theatre; we communicate risk calmly |
| **Generic SaaS blue** | indistinct; "another dashboard" |
| **Crypto/Web3 gradients** | hype, speculation, distrust |
| **Dark-only identity** | "dark dashboard vendor"; cognitive heaviness |
| **Rainbow data palettes** | noisy; contradicts calm/analytical |

---

## 7. The relationship to Arabic-first and mobile

- Colour and contrast must be tuned for **Arabic text legibility** (Arabic's stroke contrast and
  density differ from Latin).
- The palette must hold up on **mobile/TikTok screens** in bright conditions — calm but legible,
  not washed out.
- Semantic colours must remain distinguishable at small sizes and for common colour-vision
  differences.

---

## 8. Deliverables for Claude Design (colour)

- A **role-based palette** assigning values to each role above (base, surface, text, brand, growth,
  profit, confidence/evidence, action/priority, risk, neutral data).
- The **confidence/evidence scale** as a defined visual language.
- The **profit-vs-growth** distinction expressed in colour.
- Light (primary) system, with any dark variant defined as secondary.
- Accessibility + Arabic-legibility validation.

> Colour in Growmerce is a semantic instrument, not decoration. Done right, the palette quietly
> teaches the operator to read confidence, risk, growth, and profit at a glance — which is the
> intelligence made visible.
