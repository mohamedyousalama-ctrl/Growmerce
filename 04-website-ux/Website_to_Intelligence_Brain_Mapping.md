# Website → Intelligence Brain Mapping

> The spine that proves the website is not a brochure but the **user-facing interface of the
> Growmerce Intelligence Brain (20)**. Every major surface is mapped to: the user state it
> serves, the intelligence it uses, the pattern layer it draws on, the user action it drives, and
> the business outcome it supports.

This document is the contract between the UX (04), the Brain (20), and the Patterns (19). If a
surface can't be mapped here, it doesn't belong on the site.

---

## 1. The master mapping

For each surface: **user state · intelligence used (Brain) · pattern layer · user action driven ·
business outcome.**

### Homepage
- **User state:** 1 Curiosity → 2 Recognition
- **Intelligence used:** fast-judgement mode; recognition output; channel/cognition breadth
- **Pattern layer:** rotating high-recognition patterns (all families, 19)
- **User action driven:** start a diagnostic / explore a recognition
- **Business outcome:** top-of-funnel capture of attention into the cognition funnel

### Commerce Diagnostic
- **User state:** 3 Diagnosis → 4 Evidence
- **Intelligence used:** the full cognition pipeline (SENSE→DECIDE under GUARD)
- **Pattern layer:** matched + compound patterns on the visitor's inputs
- **User action driven:** complete the diagnostic → view report preview
- **Business outcome:** qualified, warm lead with a real diagnosis

### Structured Input
- **User state:** 3 Diagnosis
- **Intelligence used:** SENSE/GROUND; Signal & Evidence Hierarchy; human-in-the-loop suggestions
- **Pattern layer:** candidate patterns/competitors inferred from input
- **User action driven:** enrich context (accept/reject/edit/add); raise confidence
- **Business outcome:** richer evidence → higher-confidence findings → stronger conversion

### Pattern Recognition (cards, pattern pages)
- **User state:** 2 Recognition (cold) / 4 Evidence (warm)
- **Intelligence used:** RECOGNISE; Confidence & Trust; counter-evidence
- **Pattern layer:** the specific pattern, rendered for a human
- **User action driven:** "see if this is happening to you" → diagnostic
- **Business outcome:** self-recognition → diagnostic starts; SEO capture

### Report Preview
- **User state:** 4 Evidence → 5 Trust → 6 Action
- **Intelligence used:** VALUE→DECIDE; Confidence & Trust; Recommendation engine; GUARD
- **Pattern layer:** ranked findings (incl. compound) + recommendations
- **User action driven:** capture + go deeper / WhatsApp
- **Business outcome:** the conversion payload → consultation → Growth Ops

### Tool Pages (by outcome)
- **User state:** 2 Recognition → 3 Diagnosis (focused)
- **Intelligence used:** a focused slice of the pipeline; Confidence/abstention
- **Pattern layer:** the pattern(s) the tool detects
- **User action driven:** use the tool → deeper audit / WhatsApp
- **Business outcome:** proof of expertise → diagnostic/conversation

### Market Pages
- **User state:** 2 Recognition (by sector)
- **Intelligence used:** context gate (H6); recognition; Risk & Threat
- **Pattern layer:** sector-localised patterns
- **User action driven:** sector-scoped diagnostic / relevant tool / WhatsApp
- **Business outcome:** SEO + sector self-recognition → qualified entry

### Lead Capture
- **User state:** 6 Action → 7 Commitment
- **Intelligence used:** ACT stage timing (post-value); honesty principle
- **Pattern layer:** the findings already shown
- **User action driven:** share WhatsApp/email to continue
- **Business outcome:** captured, warm lead

### WhatsApp / Consultation CTA
- **User state:** 7 Commitment → (Growth Operations)
- **Intelligence used:** fast-judgement → deep-diagnosis bridge; Recommendation→delivery (11/12)
- **Pattern layer:** the visitor's specific findings, carried into the chat
- **User action driven:** start a context-aware conversation
- **Business outcome:** scoped Growth Operations opportunity

### Mobile / TikTok Landing
- **User state:** 1 Curiosity → 2 Recognition → 3 Diagnosis (compressed, fast)
- **Intelligence used:** fast-judgement at maximum speed
- **Pattern layer:** the single pattern the video dramatised
- **User action driven:** one-tap diagnostic + quick input → preview → WhatsApp
- **Business outcome:** convert fleeting mobile attention into qualified intent

## 2. The website as the Brain's two-mode public surface

The site renders the Brain's two cognition modes (20, §6):

```
FAST JUDGEMENT (public)              DEEP DIAGNOSIS (engagement)
homepage · tools · TikTok · quick    via WhatsApp / consultation / connected data
diagnostic                           ↓
↓  Medium-confidence findings,        High/Very-High confidence, sized impact,
   honestly hedged                    executable plan
recognition → trust → capture        confirm → scope → Growth Operations
```

The website *is* fast-judgement mode walking the visitor up the certainty ladder; the conversation
is where it hands off to deep diagnosis. The findings stay consistent across the handoff (a trust
asset).

---

## 3. Progressive intelligence reveal (what each state shows and withholds)

The Brain reveals only what each state needs, withholding the rest to power the next transition
(this is also how the AI agents decide what to surface when):

| User state | Reveals | Withholds (for later) |
|---|---|---|
| 1 Curiosity | a leak exists, generally | anything specific to them |
| 2 Recognition | the pattern, un-personalised | evidence, confidence |
| 3 Diagnosis | patterns on their inputs | full sizing, the plan |
| 4 Evidence | signals + honest confidence | execution detail |
| 5 Trust | system narrative + ruled-out | deep-diagnosis depth |
| 6 Action | the prioritised first action | the full operated plan |
| 7 Commitment | the conversation + what's next | (handoff to deep diagnosis) |

This staged reveal is the mechanism that makes the journey feel like *progressive certainty* and
keeps the visitor advancing rather than overwhelmed.

---

## 4. The intelligence contract (per surface, what must hold)

For every surface, three things must be true or the surface is broken:
1. **Grounded:** what it shows traces to real intelligence (no claim the Brain can't back) — GUARD.
2. **Honest:** confidence and limits are shown, never faked — Confidence & Trust Model.
3. **Forward:** it advances exactly one user state with one clear next action.

---

## 5. Why this mapping matters downstream

- **Claude Design (03)** designs each surface around its *state transition and intelligence
  reveal*, not its layout.
- **Claude Code** builds each surface as a *flow that moves a state and calls the right Brain
  stage*, not a static page.
- **The AI agents (07/20)** use the progressive-reveal table to know *what to surface at each
  state* — the website becomes a controllable expression of the Brain, not hardcoded copy.
- **Future SaaS** inherits the same mapping: the productised Brain surfaces are the same states,
  rendered for a returning user (see `Intelligence_Retention_and_Return_Loops.md`).
