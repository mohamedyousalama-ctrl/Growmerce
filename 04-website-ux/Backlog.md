# Backlog — Website UX

Future work for the Website UX. Candidates, not commitments. Graduates when specified enough to
hand to brand (03), build, or the AI agents.

---

## User-state journey

- **State instrumentation spec** — define the measurable proxy for each of the seven states and
  the transition-failure signals, so the live site can be diagnosed (Open Question #1).
- **Entering-state detection** — logic for each surface to detect the visitor's entering state
  (cold TikTok vs. returning user) and adapt the intelligence reveal (Open Question #2).
- **State-regression recovery patterns** — designed recoveries when a visitor drops back a state.

## Diagnostic & input

- **Suggestion-accuracy harness** — measure and improve the Spotlight-style inference (competitors,
  categories, patterns); graceful handling of confident-but-wrong suggestions (Open Question #5).
- **Input depth tuning** — find the friction/confidence sweet spot per cognition mode (#6).
- **URL/file observation depth** — specify exactly what the Brain can conclude from a live URL or
  a given file type, and the honest confidence ceiling for each.

## Report & recommendation

- **Report interaction spec** — how findings expand, evidence is inspected, and the deep-diagnosis
  path is offered, as a living surface (not a PDF).
- **Signature impact visuals** — the named visual per pattern (demand-vs-visibility, gross-vs-true
  margin, etc.) for report + homepage, to brief brand (03).

## Trust & honesty

- **Honest-confidence phrasing tests** — validate that calibrated language converts with sceptics;
  find the best Medium-confidence framing for trust + action (Open Question #7).
- **Ruled-out prominence** — calibrate how visibly "what's not wrong" appears (#8).

## Retention & return (from Intelligence_Retention_and_Return_Loops)

- **Saved profile spec** — the structured commerce profile object, persistence, and edit-on-return
  flow (ties to 13).
- **Opportunity tracker** — findings → tracked backlog with status + impact + progress.
- **Peer benchmarking** — peer dataset source, anonymisation, consent model, honest presentation
  (Open Question #11).
- **Intelligence-triggered follow-ups** — change-magnitude thresholds that justify a return
  invitation; the anti-spam line (Open Question #10).
- **Intelligence history / timeline** — the record of diagnoses, actions, outcomes.
- **Future account experience** — lightweight account holding profile/tracker/benchmarks/history;
  the SaaS seed (Open Questions #9, #17).
- **Free/paid line for returning users** — keep retention feeding (not cannibalising) Growth
  Operations (Open Question #12).

## Arabic & localisation

- **Terminology lock** — finalise core Arabic vocabulary with a native commerce writer; consistency
  spine sitewide (Open Question #13).
- **WhatsApp register** — decide whether WhatsApp/CTA flexes to a warmer, lightly-colloquial tone
  while the site stays Fusha (Open Question #14).
- **Per-region market localisation** — platform sets and language per market without fragmenting
  (Open Question #15).

## Content surfaces

- **Pattern page library** — one SEO recognition page per pattern (19), the search surface.
- **Market page set** — build out the eight market pages with sector-localised patterns.
- **TikTok lander system** — templated landers, one per video hook, with 1:1 hook-to-landing match.

## Cross-workstream handoffs

- **→ 03 Brand Identity (next):** visual system to express the command-center, ecosystem map,
  recognition cards, report, and the state journey — designed around psychology, not pages.
- **→ 06 Diagnostic:** the diagnostic UX is the web rendering of 06; align modules to the flow.
- **→ 07 AI Agents:** implement progressive intelligence reveal per state + per return visit.
- **→ 13 Input Layer:** the structured input + saved profile are the web face of 13.
- **→ 08/09 Lead Gen:** lead magnets + "which pattern costs you most"; nurture recurring users.
- **→ 12 TikTok Growth Engine:** hook→lander continuity system.
- **→ Future SaaS:** the retention/account stack as the productisation path.
