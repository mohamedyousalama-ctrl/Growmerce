# 12 — Knowledge Quality Framework

> How knowledge earns and keeps trust. A compounding knowledge base is only an asset if it stays
> *true*; an institution that accumulates unvalidated beliefs accumulates liabilities. This framework
> defines how knowledge is validated, how it gains and loses trust, how it is reviewed, and how it is
> versioned — the quality discipline of the Knowledge System.

It is the knowledge analogue of the Supply Chain's Verification and Quality Control (22): the same
institutional seriousness, applied to understanding rather than to data.

---

## 1. The quality mandate

> No knowledge object influences reasoning above the certainty its validation supports — and every
> object's certainty is earned, reviewed, and revisable.

A belief held confidently but never validated is exactly as dangerous in the Knowledge System as a
fabricated fact is in the Supply Chain. The Quality Framework exists to prevent the institution from
believing its own untested assumptions.

---

## 2. How knowledge is validated

Validation establishes that a knowledge object is *true within its stated scope*. Methods, by object
type (03):

| Object type | How it's validated |
|---|---|
| **Facts** | verification against authoritative/observable reality (re-checkable) |
| **Benchmarks** | derived from sufficient, representative data; periodically re-derived |
| **Rules / Patterns** | confirmed to recur across **independent** contexts with consistent evidence |
| **Models / Principles** | explanatory power confirmed (they predict and explain across cases) |
| **Playbooks** | validated by **outcomes** — the fix actually moved the number, repeatedly |
| **Heuristics** | useful-but-fallible by design; validated as *better-than-baseline*, with scope |
| **Lessons / Case observations** | provisional until they recur; the raw input to validation |
| **Assumptions** | explicitly *not* validated — held provisionally, flagged as such (13) |

The strongest validation is **outcome validation** (a playbook that demonstrably worked, repeatedly,
across independent contexts) — proprietary to Growmerce because it operates the fixes (links to 22, 11).

---

## 3. How knowledge gains trust (certainty rises)

An object's certainty increases when:
- it **recurs across independent contexts** (convergence — not the same context repeated);
- **outcomes confirm** its predictions/fixes;
- it is **explained by** a higher-order model/principle (we understand *why* it holds);
- it **survives challenge** (attempts to disprove it fail — the same "what would make this wrong"
  discipline as the Brain's GUARD, 20).

Rising certainty promotes the object up the hierarchy (03/11) and grants it more authority in
reasoning.

---

## 4. How knowledge loses trust (certainty falls)

An object's certainty decreases — and it may be deprecated — when:
- **outcomes contradict** it (it predicts poorly; drift, 11);
- **exceptions accumulate** until the rule's scope collapses;
- a **market shift** invalidates its conditions (07);
- a **contradiction** with a higher-certainty object is found and resolved against it (11);
- it is found to rest on **biased or insufficient** original support (a flawed benchmark, a
  non-independent sample).

Losing trust is normal and healthy — it is how the base stays true. The failure is *not* losing trust
in stale knowledge.

---

## 5. How knowledge is reviewed

Review is the institutional process that maintains quality:

| Review type | Trigger | Purpose |
|---|---|---|
| **Promotion review** | an object accrues enough support to climb a type | confirm it has earned the higher certainty (11) |
| **Drift review** | outcomes start contradicting an object | decide refine / bound / deprecate |
| **Periodic review** | scheduled, per domain | re-validate benchmarks; check currency; retire stale objects |
| **Contradiction review** | two objects conflict | resolve by hierarchy/scope/outcome (11) |
| **Challenge review** | an object is high-stakes | deliberately attempt to disprove before relying on it |

Review is owned per domain (02) — each domain has stewardship, the way a professional firm assigns
practice-area ownership. High-stakes or material changes pass human expert review (links to the
Brain's Human Override, 20): the institution does not let an automated process silently rewrite its
doctrine.

---

## 6. How knowledge is versioned

Every knowledge object is **versioned** so the institution has memory and auditability:
- **Status** — *current · under review · deprecated.*
- **Version history** — what the object said before, when and why it changed.
- **Provenance** — how it was originally established and validated (03 metadata).
- **Reasoning linkage** — which conclusions relied on which version (so a past recommendation can be
  understood in light of the knowledge that produced it).

Versioning makes the Knowledge System *auditable*: when a conclusion is questioned, the exact knowledge
(at its then-current version) that produced it can be retrieved. It also makes evolution safe —
changes are reversible and traceable, not silent overwrites.

---

## 7. Quality and confidence (the link to output)

Knowledge quality flows directly into output confidence (the Brain, 20; the Supply Chain, 22):
- A conclusion drawn from **high-certainty, validated** knowledge can be stated with high confidence.
- A conclusion drawn from **provisional or low-certainty** knowledge (assumptions, unvalidated
  heuristics) is capped lower and framed honestly.
- A conclusion that required **deprecated or contradicted** knowledge is not made.

This is the mechanism by which the institution's *honesty about what it knows* (13) becomes the
customer-facing calibrated confidence the brand promises (21). Knowledge quality is upstream of trust
language.

---

## 8. The doctrine of knowledge quality

- **Certainty is earned, never assumed.** Validation precedes authority.
- **Outcomes are the highest validator.** A playbook is trusted because it worked, repeatedly.
- **Losing trust in stale knowledge is success, not failure.**
- **Everything is reviewed, versioned, and owned.** No anonymous, immortal beliefs.
- **Assumptions are labelled, never laundered into facts.** (Bridges to Unknowns, 13.)

> The Quality Framework is what makes the Knowledge System an *institution* rather than an accumulating
> set of opinions. It is the reason the base can compound (11) without rotting — the discipline that
> keeps a growing body of knowledge *true*.
