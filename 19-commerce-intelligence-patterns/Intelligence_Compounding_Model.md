# Intelligence Compounding Model

> The reason patterns are the core asset: they make Growmerce **smarter with every
> engagement**. Industry knowledge does not compound across industries. Pattern knowledge
> does — because patterns are context-independent. This document defines the compounding loop
> that turns every client engagement into permanent, transferable intelligence.

---

## 1. Why patterns compound and categories don't

A lesson learned in a fashion engagement, stored in a fashion playbook, helps only the next
fashion client. A lesson learned about the **Review Density Gap** pattern helps *every* future
marketplace client — beauty, electronics, F&B — because the pattern is the same everywhere.

```
Category knowledge:  learning stays in its context        → linear, siloed
Pattern knowledge:   learning transfers across contexts   → compounding, networked
```

Each confirmed pattern, each validated fix, each detection refinement is an asset that pays
off across the entire future client base, not just one slice of it.

---

## 2. The compounding loop

```
        ┌─────────────────────────────────────────────────────┐
        │                                                     │
        ▼                                                     │
1. DETECT    pattern in a real engagement (06/07)             │
2. ACT       deliver the fix (Service Delivery)              │
3. MEASURE   did the signals move? did impact land?          │
4. LEARN     update the pattern's intelligence:              │
               • Track Record (Confidence Model)             │
               • which fixes worked, in which contexts       │
               • which evidence signals were most predictive │
               • which report framings drove action          │
5. ENCODE    write learning back into the library + models   │
        │                                                     │
        └─────────────────────────────────────────────────────┘
                 every engagement feeds the next
```

Each pass makes detection more accurate, fixes more proven, and reports more persuasive — for
all patterns, for all future clients.

## 3. What specifically compounds

| Asset | How it improves per engagement | Where it lives |
|---|---|---|
| **Detection accuracy** | Track Record factor rises as the pattern predicts correctly | Confidence Model |
| **Fix effectiveness** | We learn which fixes actually moved the signals | Pattern fix fields + Delivery Mapping |
| **Evidence quality** | We learn which signals are most discriminating | Evidence Framework |
| **Impact calibration** | Predicted vs. actual impact tightens the Scoring Model | Scoring Model |
| **Recognition copy** | We learn which framings trigger "that's my business" | Website / Report Usage |
| **New patterns** | Recurring un-named realities surface from the data | Backlog → library |

---

## 4. The four data exhausts every engagement produces

1. **Confirmation data** — was the pattern actually present? (calibrates confidence)
2. **Outcome data** — did the fix work? (calibrates fixes + impact)
3. **Signal data** — which evidence mattered most? (calibrates evidence)
4. **Language data** — which framing landed? (calibrates communication)

These exhausts are captured as standard at the end of every diagnostic and delivery cycle.
They are the fuel of the compounding loop — without disciplined capture, the loop stalls.

---

## 5. New-pattern discovery

Compounding also *grows the library*. When the same un-named reality appears across multiple
engagements — a recurring symptom cluster with no matching pattern — it is a candidate new
pattern. It enters `Backlog.md` as a Hypothesis, gets the full template, and is promoted
through the lifecycle as evidence accrues. The library is designed to grow **from reality**,
not from brainstorming.

```
Recurring unexplained reality → candidate pattern (Backlog) → Hypothesis → Observed → Confirmed
```

This is the anti-bloat discipline in action: new patterns earn their place by recurring in the
real world, not by sounding clever.

---

## 6. The moat

The compounding loop is Growmerce's durable advantage. A competitor can copy a playbook or a
service package. They cannot copy the **accumulated, calibrated pattern intelligence** built
from hundreds of engagements — the proven fixes, the calibrated confidence, the recognition
language that converts. The longer Growmerce operates, the wider this moat gets, and it widens
across *all* contexts at once because the unit is the transferable pattern.

```
Time + engagements → more confirmed patterns
                   → better-calibrated detection
                   → more proven fixes
                   → sharper recognition copy
                   → a compounding intelligence moat
```

---

## 7. Governance of compounding

- Every engagement closes with capture of the four data exhausts (made a delivery requirement).
- Pattern intelligence updates are reviewed before being encoded (avoid overfitting to one client).
- The Track Record factor is updated on confirmed evidence only, not anecdote.
- New candidate patterns require recurrence across **≥2 contexts** before promotion past
  Hypothesis — preserving the transferability that makes patterns valuable.
