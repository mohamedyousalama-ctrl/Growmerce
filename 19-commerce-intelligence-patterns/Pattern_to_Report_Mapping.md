# Pattern → Report Mapping

> Connects patterns to **client-facing reports**. A report is a pattern rendered as a
> diagnosis: named problem, evidence, impact, and cure. Patterns give reports their
> structure, their persuasiveness, and their "that is exactly my business" moment.

---

## 1. The report is the pattern, rendered

Every report finding is a detected pattern. The report does not invent new content — it
*renders* the pattern's template fields into a narrative a business owner reads and acts on.

| Report element | Sourced from pattern field |
|---|---|
| Finding headline | Pattern Name (sharpened for this client) |
| "What's happening" | Description + Business Symptoms |
| "How we know" | Evidence Signals (with tier) + Confidence band language |
| "What it's costing you" | Revenue Impact + Profit Impact (sized by Scoring Model) |
| "Why it matters now" | Risk Level |
| "What we'd do first" | Quick Wins |
| "The real fix" | Strategic Fixes + Operational Fixes |
| "How sure we are" | Confidence Model band, in honest language |

---

## 2. Report structure driven by scoring

The `Pattern_Scoring_Model.md` orders the report:

```
1. Headline diagnosis      → the single highest-priority pattern (the hook)
2. Stop the bleeding (P0)  → critical/high-risk patterns
3. High-leverage (P1)      → biggest upside this quarter
4. Worthwhile (P2)         → solid additional wins
5. Watchlist (P3)          → monitor, with the data we'd need to confirm
```

A report never dumps every detected pattern. It leads with the one that makes the owner say
*"that's exactly my business,"* then sequences the rest by priority — and explicitly states
what it is *not* worried about (builds credibility).

---

## 3. Report types and the patterns they feature

| Report Type | Role | Pattern usage |
|---|---|---|
| **Free self-diagnosis** (website/lead magnet) | Recognition + lead capture | 1–3 high-recognition patterns at Possible/Probable confidence; impact teased, full fix gated |
| **Standard audit** (paid entry) | Prioritised diagnosis | Full ranked pattern set at Probable confidence; quick wins included |
| **Deep diagnostic** | Confirmed, evidenced diagnosis | All patterns at Confirmed confidence with full evidence trails and sized impact |
| **Ongoing scorecard** (retained client) | Track pattern resolution over time | Shows patterns moving from detected → fixed; new patterns as they emerge |

The same pattern library powers all four — depth of evidence and fix detail differ, the
patterns do not.

---

## 4. The persuasion mechanics

Patterns make reports persuasive because they are:
- **Recognisable** — the Description + Symptoms trigger self-recognition.
- **Evidenced** — Evidence Signals + Confidence band prove it's not generic.
- **Quantified** — Revenue/Profit Impact + Scoring size the prize.
- **Actionable** — Quick Wins give an immediate, low-risk next step.
- **Honest** — Counter Evidence and Confidence language build trust by admitting uncertainty.

The most powerful single report asset is the **impact-gap visual** each pattern implies (e.g.
demand-vs-visibility, gross-vs-true-margin, peak-cohort decay). Each pattern's Report Usage
field names its signature visual.

---

## 5. Tone and honesty rules

- State confidence honestly using the Confidence Model language ("the data clearly shows" vs.
  "we strongly suspect"). Never overclaim.
- Always pair a problem with a path (Quick Win) so the report empowers rather than alarms.
- Size impact in ranges, not false precision.
- Name what's working, not only what's broken — credibility comes from balance.

---

## 6. Loop back

Which report framings drive action (acceptance, engagement, conversion to paid work) feeds
the `Intelligence_Compounding_Model.md`: we learn which pattern *renderings* land, not just
which patterns are true. Over time the library's Report Usage fields sharpen toward the
framings that consistently produce the "that's exactly my business" reaction.
