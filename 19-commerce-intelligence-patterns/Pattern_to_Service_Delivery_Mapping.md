# Pattern → Service Delivery Mapping

> Connects patterns to **Growth Operations delivery**. A detected pattern isn't intelligence
> until it becomes work that fixes it. This mapping turns each pattern's fixes into reusable
> delivery motions, so Growth Ops stops reinventing solutions and starts executing proven
> playbooks.

---

## 1. The principle: patterns make delivery reusable

Before patterns, delivery was bespoke — every engagement designed from scratch. With patterns,
each pattern's **Quick Wins / Strategic Fixes / Operational Fixes** become a **standard fix
playbook**. Delivery becomes: detect pattern → pull its fix playbook → execute as a sprint →
measure resolution. The same fix transfers across industries because the pattern does.

```
Pattern detected → Fix playbook (from pattern's fix fields) → Delivery sprint → Resolution measured → Compounding
```

---

## 2. From pattern fixes to delivery sprints

Each pattern's three fix tiers map to a delivery cadence:

| Pattern fix tier | Delivery motion | Timeframe |
|---|---|---|
| **Quick Wins** | Week-one sprint — fast, visible, trust-building | Days |
| **Strategic Fixes** | Core engagement work — the durable solution | Weeks–quarter |
| **Operational Fixes** | Embedded process/system change — stops recurrence | Ongoing |

Standard engagement shape: open with one or two **Quick Win** sprints (earn trust fast), then
the **Strategic Fix** as the main body of work, then install **Operational Fixes** so the
pattern doesn't return after Growmerce leaves.

## 4. Sequencing across multiple detected patterns

When a diagnostic surfaces several patterns, delivery is sequenced by the Scoring Model plus
**dependencies**:
- Fix upstream patterns first (e.g. *Discovery Fails* before optimising conversion on traffic
  that doesn't exist).
- Front-load one Quick Win for early trust.
- Prefer fixes that build compounding equity (owned audience, organic rank) where priority is
  close.

The Recommendation Agent (07) produces this sequence; Growth Ops executes it.

---

## 5. Measuring resolution

Each delivery sprint closes the loop by re-checking the pattern's Evidence Signals:
- Did the signals move in the predicted direction?
- Did confidence in the pattern drop (i.e. is it resolved)?
- Did the predicted Revenue/Profit Impact materialise?

Resolution data feeds the `Intelligence_Compounding_Model.md` — proving which fixes work, for
which patterns, in which contexts. This is how Growmerce's delivery gets measurably better
over time instead of just busier.

---

## 6. Why this protects margin and quality

Reusable fix playbooks mean:
- Faster delivery (less bespoke design).
- More consistent quality (proven motions).
- Easier scaling of the team (playbooks train people).
- Defensible pricing (outcomes, not hours).

Patterns turn Growth Operations from a labour business into a leveraged, compounding one —
without changing the business model. Same company, sharper execution.
